                    TERMINAL SCROLLBAR - QUICK REFERENCE

FILE MODIFIED: src/ui/components/terminal.html

KEY CHANGES:
================================================================================

1. CSS CHANGES (Force Scrollbar Display)
   Before: overflow: auto          (only shows when needed)
   After:  overflow-y: scroll      (always show scrollbar)
   Impact: Scrollbar always visible, users know it's available

2. SCROLLBAR STYLING (Webkit - Chrome, Edge, Safari)
   - Width: 14px (reasonable size)
   - Color: Blue gradient (rgba(100,160,255) to rgba(60,120,200))
   - Track: Dark background (rgba(20,20,20,0.8))
   - Thumb: Gradient with 40px minimum height
   - States: Normal, Hover (brighter), Active (brightest)

3. FIREFOX FALLBACK
   - Uses: scrollbar-width: thin
   - Color: rgba(80,150,240) on rgba(20,20,20)
   - Appearance: Simplified but consistent

4. JAVASCRIPT ENHANCEMENTS
   - Added _ensureScrollbarVisible() function
   - Called after write operations
   - Guarantees scrollbar stays visible

5. TERMINAL CONFIG UPDATES
   - Scrollback increased: 3000 → 5000 lines
   - Mouse wheel mode: 'line' (enabled wheel scrolling)
   - Auto-scroll: true on both input and output

TECHNICAL DETAILS
================================================================================

Layout Hierarchy:
  html/body (100% height, overflow: hidden)
    ↓
  #terminal-container (flex column, 100%)
    ↓
  .xterm-viewport (flex: 1, overflow-y: scroll)
    ↓
  .xterm-screen (height: auto, grows with content)

CSS Priority:
  1. JavaScript inline styles (highest)
  2. !important rules
  3. Regular CSS rules
  4. Browser defaults (lowest)

Scrollbar Implementation:
  - ::-webkit-scrollbar (Chrome, Edge, Safari, Opera)
  - scrollbar-width/color (Firefox fallback)
  - Pure CSS (no JavaScript polling)
  - Hardware accelerated

================================================================================
FREQUENTLY ASKED QUESTIONS
================================================================================

Q: Why is scrollbar always visible even with small content?
A: Design choice. Always-visible = users know it's available.
   Standard practice in modern IDEs.

Q: Why blue gradient instead of solid color?
A: Matches Cortex IDE theme. Gradient is more visually appealing
   and provides better depth perception.

Q: Why 5000 lines scrollback?
A: Balance between history (for debugging) and memory (no bloat).
   Typical for terminal applications.

Q: Why force scrollbar display with JavaScript?
A: Guarantees scrollbar persists even with dynamic content changes.
   Prevents edge cases where CSS might fail.

Q: Will this break on new xterm.js versions?
A: Unlikely. Uses standard CSS and HTML attributes.
   Check .xterm-viewport class exists if updating.
