Since I cannot directly read the files due to tool limitations, I will provide a detailed architectural overviewand Mermaid diagramsbased on the project structure and memory records. This will help you understand how the Cortex IDEproject works.

A Vision Agentfor OCR and object detection.
A file-based memory systemfor persistence.
A PyQt6-based UIfor user interaction.
Signal-based communicationbetween agents and the UI.
Tool integrationfor code intelligence, debugging, and automation.
Vision Agent: Specialized for image processing (OCR, object detection).
Coordinator Agent: Manages task delegation and agent collaboration.
Signal/Slot Mechanism: Enables communication between UI and agents.
Tool Registry(tool_registry.py): Manages available tools for agents.
Context Manager(context.py): Maintains conversation and project context.
Task Manager(tasks.py): Tracks and executes tasks.
Memory Types: User, feedback, project, and reference memories.

sequenceDiagram

    participant UI
    participant MainAgent
    participant Coordinator

    User->>UI: Sends Query
    UI->>MainAgent: Emits Signal (e.g., user_question_requested)
    MainAgent->>Coordinator: Delegates Task
    Coordinator->>VisionAgent: Assigns Vision Task
    VisionAgent->>Coordinator: Returns->>MainAgent: Aggregates Results
    MainAgent->>UI: Updates UI with Response
    UI->>User: Displays Result
